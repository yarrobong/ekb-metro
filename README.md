# Метро Екатеринбурга

Мобильное PWA-приложение для просмотра времени до следующего поезда Екатеринбургского метрополитена.

## Технологии

- React
- TypeScript
- Vite
- Tailwind CSS
- Zustand
- Zod
- vite-plugin-pwa

## Локальный запуск

```bash
npm install
npm run dev
```

## Проверка проекта

```bash
npm run typecheck
npm run lint
npm run format:check
npm run test:run
npm run build
```

## Production-предпросмотр

```bash
npm run build
npm run preview
```

## Данные расписания (Этап 2)

Добавлен полный и валидируемый набор данных о станциях, направлениях и расписании метрополитена.

- `src/domain/metro` содержит типы и Zod-схемы;
- `src/data` содержит списки станций, перегонов и само расписание;
- Для проверки цельности данных используется специальный скрипт `scripts/validate-metro-data.ts`.

Для проверки данных выполните:

```bash
npm run validate:data
```

Для добавления или обновления расписания:

1. Замените или дополните данные в `src/data/schedule.ts`.
2. Обновите дату проверки в `src/data/metadata.ts`.
3. Обязательно запустите автоматическую проверку и тесты:

```bash
npm run validate:data
npm run test:run
```

## ЭТАП 3. ИНТЕРАКТИВНЫЙ ВЫБОР СТАНЦИИ И НАПРАВЛЕНИЯ (Реализован)

Создан полностью рабочий пользовательский сценарий выбора текущей станции и направления движения.
Теперь приложение содержит:

- Интерактивную схему линии метро
- Обработку выбора станции (с автоматическим направлением для конечных станций)
- Модальное окно (BottomSheet) для выбора направления на промежуточных станциях
- Временный виджет на главном экране (Поезда), где отображаются: выбранная станция, возможность смены направления, следующая станция и карточка-заглушка расписания.
- Состояние selections сохраняется только в текущей сессии (Zustand).

Для проверки E2E и компонентных тестов, выполните:

```bash
npm run test:run
```

## ЭТАП 4. ЛОГИКА БЛИЖАЙШЕГО ПОЕЗДА И ЖИВОЙ ТАЙМЕР (Реализован)

Реализована основная функция приложения — определение ближайшего поезда для выбранной станции и направления.

- Создан единый сервис времени, все расчеты производятся в часовом поясе `Asia/Yekaterinburg`.
- Автоматически выбирается тип расписания (будни или выходные, с учетом специальных дат).
- Основной экран отображает главный таймер и список из 4-х следующих поездов (относительное время пересчитывается без дублирования первого).
- Отработаны три статуса: ожидание, "Поезд приближается" (< 30с) и "Поезд прибывает" (окно 15 секунд).
- Настройки отображения секунд корректно влияют на формат времени (от полного таймера до русских текстов "4 минуты").
- Написаны unit-тесты логики времени и расписания.

Особые случаи: полночный переход, закрытое метро и время до открытия будут добавлены на этапе 5.

## ЭТАП 8. PWA, установка и офлайн-режим (Реализован)

Приложение подготовлено как устанавливаемое PWA:

- подключён `vite-plugin-pwa` со стратегией `generateSW`;
- настроены `manifest`, `standalone`, `theme-color`, `start_url` и `scope` под GitHub Pages;
- добавлены PWA-иконки, Apple Touch Icon и `404.html` для fallback;
- после первой успешной загрузки приложение может открываться офлайн;
- есть карточка предложения установки, уведомление о готовности офлайн-режима и баннер обновления;
- в настройках доступна ручная проверка обновлений.

Проверять PWA нужно на production-сборке:

```bash
npm install
npm run build
npm run preview
```

Рекомендуемая ручная проверка:

1. Открыть приложение в preview или тестовой публикации GitHub Pages.
2. Проверить установку на Android/desktop через системный prompt.
3. Проверить инструкцию и ручную установку на iPhone.
4. После первой загрузки отключить интернет и убедиться, что приложение открывается офлайн.
5. Собрать новую версию и проверить появление предложения обновления.
